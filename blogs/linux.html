Bash

Bash Command Cheat Sheet

Basic Bash Commands

echo

Displays a line of text or a variable's value.

echo "Hello, World!" # Output text

ls

Lists files and directories in the current directory.

ls # List files and directories

cd

Changes the current directory.

cd /path/to/directory # Change to the specified directory

Advanced Bash Commands

grep

Searches for a specified pattern in a file or input.

grep "pattern" file.txt # Search for pattern in file.txt

find

Finds files and directories in a specified location.

find /path/to/search -name "filename" # Find file by name

top

Displays real-time system information, including processes and resource usage.

top # Display real-time process stats

chmod

Changes the file permissions for a file or directory.

chmod 755 file.txt # Set read/write/execute permissions for owner, and read/execute for others

kill

Kills a process by its PID (process ID).

kill  # Terminate a process by its PID

tar

Archives and extracts files from tarballs.

tar -cvf archive.tar files/ # Create a tarball
tar -xvf archive.tar # Extract a tarball

df

Displays disk space usage for file systems.

df -h # Display disk usage in a human-readable format

ps

Displays information about the current running processes.

ps aux # List all running processes

history

Displays the history of commands executed in the terminal.

history # Show the command history


Windows PowerShell

PowerShell Cheat Sheet

Basic PowerShell Commands

Get-Help

Shows help for a command.

Get-Help  # Shows help for a command

Get-Command

Find a command by name.

Get-Command  # Find a command by name

Get-Process

Lists running processes on the system.

Get-Process # Lists running processes on the system

Advanced PowerShell Commands

Get-EventLog

Get events from the event log. Useful for checking system events and issues.

Get-EventLog -LogName Application -Newest 10 # Get the 10 most recent application logs

Set-ExecutionPolicy

Change the script execution policy on your machine. Useful for allowing or restricting script execution.

Set-ExecutionPolicy RemoteSigned # Allows running scripts created locally, but blocks unsigned scripts from the internet

Get-ItemProperty

Gets the properties of a specified item, such as registry entries or file properties.

Get-ItemProperty -Path "HKCU:\Software\Microsoft\Windows\CurrentVersion\Explorer" # Get properties from the registry

Get-WmiObject

Get management information from local or remote machines, such as hardware information or system configurations.

Get-WmiObject -Class Win32_OperatingSystem # Get OS information

Invoke-Command

Run commands on remote computers, useful for remote administration.

Invoke-Command -ScriptBlock { Get-Process } -ComputerName Server01 # Run Get-Process on a remote server

Basic CMD Commands

Changes the current directory.

Advanced CMD Commands

ping

Sends network packets to a specific host to test connectivity.

ipconfig

Displays network adapter configuration details like IP address and subnet mask.

ipconfig /all # Display detailed network configuration

tasklist

Lists all running processes on your machine.

taskkill

Kills or terminates a running process by name or PID (Process ID).

taskkill /PID  # Kill a process by its PID
taskkill /IM  # Kill a process by its name

netstat

Displays active network connections and listening ports on your computer.

netstat -ano # Show active connections and their associated PIDs

shutdown

Shuts down or restarts the system with a specified timeout and options.

shutdown /s /t 30 # Shut down the system after a 30-second delay
shutdown /r /t 0  # Restart the system immediately

MATLAB

MATLAB Command Cheat Sheet

Basic MATLAB Commands

disp

Displays a message or variable in the Command Window.

disp('Hello, MATLAB') % Outputs 'Hello, MATLAB' in the command window

length

Returns the number of elements in an array or matrix.

len = length([1, 2, 3, 4, 5]) % Returns 5

input

Prompts the user for input in the command window.

name = input('Enter your name: ', 's') % Takes string input from user

Advanced MATLAB Commands

Matrix Multiplication

Performs matrix multiplication. Use '*' for standard multiplication.

A = [1, 2; 3, 4];
B = [5, 6; 7, 8];
C = A * B; % Matrix multiplication

Anonymous Function

Defines a function without the need for a separate file.

f = @(x) x^2 + 2*x + 1; % Anonymous function for a quadratic equation
result = f(3); % Evaluate at x = 3, returns 16

for Loop

Executes a block of code multiple times using a specified range.

for i = 1:5
    disp(['Iteration: ', num2str(i)])
end % Loops through values 1 to 5

plot

Creates a 2D plot of data points.

x = 0:0.1:10;
y = sin(x);
plot(x, y) % Plots the sine wave

function Definition

Defines a reusable function in MATLAB.

function result = square(x)
    result = x^2;
end % Defines a function to square a number
result = square(4); % Returns 16

load

Loads data from a .mat file into the workspace.

load('datafile.mat') % Loads variables from the file 'datafile.mat'

save

Saves workspace variables to a .mat file.

save('output.mat', 'A', 'B') % Saves variables A and B to 'output.mat'

error Handling (try-catch)

Handles errors that occur during the execution of code.

try
    result = 10 / 0; % Deliberate error (division by zero)
catch
    disp('An error occurred during execution');
end


Python

Python Command Cheat Sheet

Basic Python Commands

Print

Prints a message or variable to the console.

print("Hello, World!") # Output a message to the console

len

Returns the length of an object (string, list, etc.).

len("Hello") # Returns 5
len([1, 2, 3]) # Returns 3

input

Accepts user input from the console.

name = input("Enter your name: ") # Get user input and store it in 'name'

Advanced Python Commands

List Comprehension

A concise way to create lists based on existing lists or other iterables.

squared = [x**2 for x in range(5)] # Returns [0, 1, 4, 9, 16]

Lambda Function

A small anonymous function defined with the lambda keyword.

multiply = lambda x, y: x * y # Multiplies two numbers
result = multiply(3, 4) # Returns 12

Map

Applies a function to all items in an iterable (e.g., list or tuple).

numbers = [1, 2, 3, 4]
squared_numbers = map(lambda x: x**2, numbers) # Applies the lambda function to each element
print(list(squared_numbers)) # Output: [1, 4, 9, 16]

Filter

Filters elements from an iterable based on a function.

numbers = [1, 2, 3, 4, 5]
even_numbers = filter(lambda x: x % 2 == 0, numbers) # Filters even numbers
print(list(even_numbers)) # Output: [2, 4]

Try-Except

Used to handle exceptions (errors) in Python.

try:
    result = 10 / 0
except ZeroDivisionError:
    print("Cannot divide by zero!") # Handle division by zero error

Decorators

A function that wraps another function to modify its behavior.

def decorator(func):
    def wrapper():
        print("Before function call")
        func()
        print("After function call")
    return wrapper

@decorator
def greet():
    print("Hello!")

greet() # Output: Before function call
        #         Hello!
        #         After function call

Class Definition

Defines a class with attributes and methods in Python.

class Dog:
    def __init__(self, name, breed):
        self.name = name
        self.breed = breed

    def bark(self):
        print(f"{self.name} says woof!")

my_dog = Dog("Buddy", "Golden Retriever")
my_dog.bark() # Output: Buddy says woof!

File I/O

Read from and write to files in Python.

with open("file.txt", "w") as f:
    f.write("Hello, Python!")
with open("file.txt", "r") as f:
    print(f.read()) # Output: Hello, Python!

Nmap Command Cheat Sheet

Basic Nmap Commands

Scan a Single Host

Scans a single IP address or hostname.

nmap 192.168.1.1  # Scan a single host

Scan a Range of IPs

Scan a range of IP addresses.

nmap 192.168.1.1-50  # Scan a range of IPs from 192.168.1.1 to 192.168.1.50

Scan a Subnet

Scan an entire subnet using CIDR notation.

nmap 192.168.1.0/24  # Scan all hosts in the 192.168.1.0 subnet

Advanced Nmap Commands

Service Version Detection

Detect service versions running on open ports.

nmap -sV 192.168.1.1  # Detect service versions for open ports on the target

Operating System Detection

Detect the operating system of the target host.

nmap -O 192.168.1.1  # Detect the OS of the target host

Aggressive Scan

Perform an aggressive scan that includes version, OS, and script scanning.

nmap -A 192.168.1.1  # Perform an aggressive scan with OS and version detection

TCP SYN Scan

Perform a TCP SYN scan (stealth scan) to check for open ports.

nmap -sS 192.168.1.1  # Perform a TCP SYN scan (stealth scan)

UDP Scan

Scan for open UDP ports on a target.

nmap -sU 192.168.1.1  # Scan for open UDP ports on the target

Scan Specific Ports

Scan specific ports on the target.

nmap -p 22,80,443 192.168.1.1  # Scan ports 22, 80, and 443 on the target

Scan Using a Script

Use Nmap's scripting engine to run specific scripts on a target.

nmap --script=http-enum 192.168.1.1  # Run the http-enum script to enumerate web server info

Save Scan Results to a File

Save the results of your scan to a file (XML, JSON, or plain text).

nmap -oN scan_results.txt 192.168.1.1  # Save the results to a plain text file

Wireshark Command Cheat Sheet

Basic Wireshark Commands

Start Wireshark in Capture Mode

Start Wireshark to capture live packets on the default interface.

wireshark  # Start Wireshark in capture mode

Capture Packets on Specific Interface

Start capturing on a specific network interface.

wireshark -i eth0  # Capture packets on eth0 interface

Capture Packets with a Filter

Capture packets with a specific filter (e.g., TCP traffic on port 80).

wireshark -f "tcp port 80"  # Capture only TCP packets on port 80

Advanced Wireshark Commands

Capture and Save Packets

Capture packets and save them to a file for later analysis.

wireshark -i eth0 -w capture_file.pcap  # Capture on eth0 and save to capture_file.pcap

Read a Saved Capture File

Open and analyze a saved capture file.

wireshark capture_file.pcap  # Open a previously saved capture file

Capture Packets with a Display Filter

Capture packets and apply a display filter to limit what is shown (e.g., filter by IP).

wireshark -Y "ip.addr == 192.168.1.1"  # Filter packets by IP address (192.168.1.1)

Analyze Traffic with Statistics

Use built-in statistics to analyze network traffic.

wireshark -z io,stat,0  # Display I/O graph statistics

Use a Custom Capture Filter

Set up a custom filter to capture specific types of packets.

wireshark -f "src net 192.168.1.0/24 and dst port 443"  # Capture packets from 192.168.1.0/24 to port 443

Export Packets to CSV

Export captured packets to a CSV file.

tshark -r capture_file.pcap -T fields -e frame.number -e ip.src -e ip.dst -e ip.proto -e tcp.port > packets.csv  # Export packet details to CSV

Save Only Specific Packet Types

Capture and save specific packet types (e.g., only HTTP packets).

wireshark -i eth0 -f "tcp port 80" -w http_capture.pcap  # Save only HTTP packets to http_capture.pcap

SQLmap Command Cheat Sheet

Basic SQLmap Commands

Run SQLmap on a URL

Run SQLmap on a URL to detect SQL injection vulnerabilities.

sqlmap -u "http://example.com/vulnerable.php?id=1"  # Run SQLmap on the provided URL

Use GET Method for Requests

Use the GET method instead of POST to send the request.

sqlmap -u "http://example.com/vulnerable.php?id=1" --method=GET  # Use GET method for the request

Enumerate Database Users

Enumerate the database users using SQLmap.

sqlmap -u "http://example.com/vulnerable.php?id=1" --users  # Enumerate database users

Advanced SQLmap Commands

Use a Custom HTTP Header

Use custom HTTP headers for the request (e.g., user-agent, referer).

sqlmap -u "http://example.com/vulnerable.php?id=1" --headers="User-Agent: Mozilla/5.0"  # Use custom HTTP headers

Dump Database Tables

Dump all tables from the database using SQLmap.

sqlmap -u "http://example.com/vulnerable.php?id=1" --dump  # Dump all tables from the database

Force Specific Injection Technique

Force SQLmap to use a specific injection technique (e.g., UNION-based).

sqlmap -u "http://example.com/vulnerable.php?id=1" --technique=U  # Force UNION-based injection

Perform a Blind SQL Injection Test

Perform a blind SQL injection test to identify vulnerabilities.

sqlmap -u "http://example.com/vulnerable.php?id=1" --technique=B  # Perform blind SQL injection test

Specify the Database Type

Specify the database management system (DBMS) type (e.g., MySQL, PostgreSQL).

sqlmap -u "http://example.com/vulnerable.php?id=1" --dbms=MySQL  # Specify MySQL as the DBMS

Enumerate Database Schema

Enumerate the database schema (tables, columns, etc.) using SQLmap.

sqlmap -u "http://example.com/vulnerable.php?id=1" --schema  # Enumerate the database schema

SQLmap with Proxy for Traffic Interception

Route traffic through a proxy (e.g., Burp Suite) for interception.

sqlmap -u "http://example.com/vulnerable.php?id=1" --proxy="http://127.0.0.1:8080"  # Use proxy for traffic interception
